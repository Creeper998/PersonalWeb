## 页面层（路由）


 ### 负责“路由映射+页面组装”  ---> 获取数据，然后渲染
   
1.**路由映射**
- 根据url 显示对应的页面
  - xxxx.XX/home ; xxxx.xx/project 
  
2.**页面组装**
- 组合多个组件完成完整页面
- 决定页面包含哪些组件、如何布局  -->显示的核心

3.**获取数据（从服务器、api或数据库等）**
- 由页面层发起数据请求
- 获取数据后，通过props传给组件
- 组件层不直接请求数据，只负责显示

4.**渲染**
- 将获取的数据通过props传递给组件
- 数据流向：API/数据库等 → 页面层 → props → 组件层 → 界面显示

#### 总结Web显示的流程
```
用户访问 /home
↓
页面层：识别路由 → 获取数据 → 组装组件 → 渲染页面
↓
组件层：接收 props → 显示界面
 ```
